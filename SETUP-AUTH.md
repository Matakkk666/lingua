# Настройка авторизации (email-коды через Supabase)

Это пошаговое руководство описывает, как подключить реальную авторизацию по email-коду (OTP) к сайту Lingua.

---

## Что вы получите

- Пользователь вводит email → на почту приходит 6-значный код
- Пользователь вводит код → входит на сайт
- Сессия сохраняется (при перезагрузке страницы не нужно входить заново)
- Кнопка «Sign out» для выхода
- Checkout защищён: чтобы оформить заказ, нужно войти

---

## Шаг 1. Регистрация в Supabase

1. Откройте **https://supabase.com** и нажмите **«Start your project»**
2. Войдите через GitHub или email
3. Нажмите **«New project»**
4. Заполните:
   - **Name:** `lingua-school` (любое имя)
   - **Database Password:** придумайте пароль (сохраните его!)
   - **Region:** выберите ближайший к вам регион
5. Нажмите **«Create new project»** и подождите 1–2 минуты

---

## Шаг 2. Получение ключей API

1. В левом меню Supabase откройте **Project Settings** (иконка шестерёнки внизу)
2. Перейдите в раздел **API**
3. Найдите два значения:

   - **Project URL** — выглядит как `https://abcdexyz.supabase.co`
   - **anon public key** — длинная строка, начинается с `eyJ...`

   > **Важно:** используйте именно `anon` ключ, а НЕ `service_role`. Anon-ключ безопасен для фронтенда — он ограничен правами, которые вы настраиваете.

---

## Шаг 3. Вставка ключей в код

1. Откройте файл **`assets/js/auth.js`** в редакторе
2. Найдите в самом верху две строки:

```js
const SUPABASE_URL = 'https://YOUR-PROJECT.supabase.co';
const SUPABASE_ANON_KEY = 'YOUR-ANON-KEY-HERE';
```

3. Замените их на ваши значения:

```js
const SUPABASE_URL = 'https://abcdexyz.supabase.co';
const SUPABASE_ANON_KEY = 'eyJhbGciOiJIUzI1NiIsInR5cCI6...ваш_ключ';
```

4. Сохраните файл

---

## Шаг 4. Проверка Email OTP (включён по умолчанию)

Supabase включает email-OTP по умолчанию, но давайте проверим:

1. В левом меню Supabase откройте **Authentication** → **Providers**
2. Найдите **Email** — он должен быть включён (зелёный переключатель)
3. Убедитесь, что опция **Enable email sign-ins** включена

---

## Шаг 5. Настройка шаблона письма (чтобы приходил код)

По умолчанию Supabase отправляет письмо с 6-значным кодом. Но давайте проверим и кастомизируем:

1. Откройте **Authentication** → **Email Templates**
2. Выберите вкладку **«Magic Link»** (именно этот шаблон используется для OTP)
3. Убедитесь, что в шаблоне есть переменная **`{{ .Token }}`** — это и есть 6-значный код

Пример хорошего шаблона:

```html
<h2>Вход в Lingua</h2>
<p>Ваш код для входа:</p>
<h1 style="font-size: 32px; letter-spacing: 8px; color: #9D7BC4;">{{ .Token }}</h1>
<p>Код действителен 1 час.</p>
<p>Если вы не запрашивали код — просто проигнорируйте это письмо.</p>
```

4. Нажмите **Save**

> **Примечание:** `{{ .Token }}` — это 6-значный OTP-код. `{{ .ConfirmationURL }}` — это magic link (ссылка для входа в один клик). Можно оставить оба или только код.

---

## Шаг 6. Тестирование

1. Откройте сайт в браузере (`index.html`)
2. Нажмите **«Sign in»** в шапке
3. Введите свой email → нажмите **«Send login code»**
4. Проверьте почту — должно прийти письмо с 6-значным кодом
5. Введите код в поле → нажмите **«Verify & sign in»**
6. Если всё верно, появится экран «You're in!» и шапка изменится на ваш email + «Sign out»

### Частые проблемы

| Проблема | Решение |
|---|---|
| Письмо не приходит | Проверьте папку «Спам». На бесплатном тарифе Supabase есть задержка до 1 минуты |
| «Setup needed» в модалке | Вы не вставили ключи в `auth.js` (Шаг 3) |
| «Too many attempts» | На бесплатном тарифе есть лимит: ~4 письма в час. Подождите или используйте другой email |
| «That code is invalid» | Код истёк (живёт 1 час) или введён с ошибкой. Нажмите «Resend code» |

---

## Шаг 7 (опционально). Свой SMTP для продакшена

На бесплатном тарифе письма приходят от `noreply@mail.supabase.com` с ограничением скорости. Для реального проекта подключите свой SMTP:

1. Откройте **Authentication** → **Settings** → **SMTP Settings**
2. Включите **Custom SMTP**
3. Заполните данные вашего почтового провайдера:
   - **Host:** например `smtp.gmail.com`, `smtp.yandex.ru`, `smtp.mailgun.org`
   - **Port:** `587` (TLS) или `465` (SSL)
   - **Username:** ваш email
   - **Password:** пароль приложения (не основной пароль!)
   - **Sender email:** с какого адреса отправлять
4. Нажмите **Save**

### Пример для Gmail

1. Включите двухфакторную аутентификацию в Google Аккаунте
2. Создайте пароль приложения: Google Account → Security → 2-Step Verification → App passwords
3. Используйте этот пароль (16 символов) в Supabase SMTP settings

## Как это работает (технически)

```
Пользователь вводит email
        │
        ▼
auth.js вызывает supabase.auth.signInWithOtp({ email })
        │
        ▼
Supabase отправляет письмо с 6-значным кодом ({{ .Token }})
        │
        ▼
Пользователь вводит код
        │
        ▼
auth.js вызывает supabase.auth.verifyOtp({ email, token, type: 'email' })
        │
        ▼
Supabase проверяет код → создаёт сессию (JWT в localStorage)
        │
        ▼
onAuthStateChange срабатывает → UI обновляется (шапка, доступ к checkout)
```

## Безопасность

- **anon-ключ** безопасен для фронтенда — он работает только в рамках RLS-политик вашей базы
- **service_role-ключ** НИКОГДА не вставляйте в фронтенд — он даёт полный доступ к базе
- Сессия хранится в `localStorage` как JWT-токен
- Для реального продакшена настройте **Row Level Security (RLS)** в Supabase, если используете базу данных
